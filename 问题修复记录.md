# Hugo 页面渲染问题修复记录

## 问题描述

以下 Markdown 文件在 Hugo 构建后没有生成对应的 HTML 页面：

- `肖秀荣政治1000题解析.md`
- `肖秀荣政治1000题解析_v2.md`
- `肖秀荣政治1000题解析_完整版.md`
- `肖秀荣政治1000题完整解析_深度版.md`
- `26年肖秀荣政治1000-试题册_by_PaddleOCR-VL-1.5.md`

同时，`/tags/` 页面内容为空。

## 问题诊断过程

### 1. 初步检查

- 文件 frontmatter 格式正确
- 文件编码为 UTF-8，无 BOM
- Hugo list 命令可以识别这些文件
- 构建过程无错误信息

### 2. 关键发现

通过对比测试发现：
- 从 math 目录复制的文件可以正常渲染
- 新创建的测试文件无法渲染
- 使用 `--buildDrafts --buildFuture` 参数后，所有文件都能渲染

### 3. 根本原因

**时区问题** - Hugo 使用 UTC 时间判断文章是否为"未来"文章：

- 文件日期：`2026-04-02`（北京时间）
- 当前时间：2026-04-02 凌晨（北京时间）= 2026-04-01 晚上（UTC）
- Hugo 判断：文件日期 > 当前 UTC 日期 = 未来文章
- 默认行为：不渲染未来文章

## 修复方案

### 修改 hugo.toml

在配置文件中添加 `buildFuture = true`：

```toml
baseURL = 'https://cs.cloudsx.top/'
languageCode = 'zh-cn'
title = '我的考研复习笔记'
theme = 'hugo-book'

disablePathToLower = true
enableGitInfo = true
timeout = '300s'
buildFuture = true  # 添加这一行

[markup]
[markup.goldmark.renderer]
  unsafe = true

[markup.tableOfContents]
  startLevel = 1

[params]
  BookTheme = 'light'
  BookSearch = true
  BookKaTeX = true
  BookSection = '*'
```

## 关于 Tags 页面

Tags 页面为空是**正常行为**，因为没有任何内容文件使用了 `tags` frontmatter。

如需使用 tags，请在内容文件的 frontmatter 中添加：

```yaml
---
title: "文章标题"
date: 2026-04-02
draft: false
tags: ["政治", "肖秀荣", "考研"]
---
```
